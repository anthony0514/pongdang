# 퐁당 앱스토어 미리보기 및 스크린샷 기획안

## 목적
- 앱스토어 `미리보기 및 스크린샷` 영역에서 퐁당의 핵심 가치를 짧고 직관적으로 전달한다.
- 첫 3장은 기능 설명보다도 "왜 이 앱이 필요한지"를 먼저 이해시키는 흐름으로 간다.
- 이후 장면에서 지도 저장, 함께 쓰기, 방문 기록, 캘린더 회고, 설정/공유 기능을 자연스럽게 보여준다.

## 전체 방향
- 타깃: 커플, 친구, 가족처럼 작은 그룹 단위로 장소를 함께 기록하고 싶은 사용자
- 핵심 메시지: `둘이 함께 저장하고, 함께 다녀오고, 함께 추억하는 지도 일기`
- 비주얼 톤: 조용한 호수 같은 블루 계열, 감정은 따뜻하고 화면은 정리된 느낌
- 카피 원칙:
  - 한 장당 핵심 문구는 짧게
  - 설명 문구는 1줄 이내
  - 기능 나열보다 사용 장면 중심

## 권장 스크린샷 수
- 최소 5장
- 권장 6장
- 여유가 있으면 7장까지 확장 가능

## 권장 순서
1. 함께 쓰는 장소 지도
2. 가고 싶은 곳 저장
3. 방문 후 기록 남기기
4. 날짜별로 추억 돌아보기
5. 스페이스로 함께 관리
6. 외부 지도 앱까지 바로 연결

---

## 페이지 1
### 역할
- 앱의 정체성을 한 번에 설명하는 대표 장면

### 메인 카피
`우리 둘만의 장소 지도를 만들어보세요`

### 서브 카피
`가고 싶은 곳부터 다녀온 추억까지, 한 지도에 함께 기록해요`

### 추천 화면
- `MapView`
- 핀이 여러 개 보이는 홈 지도 화면
- 미방문 핀과 방문 완료 핀이 함께 보이도록 구성

### 화면 연출 가이드
- 지도 전체가 보이되 핀이 적당히 분포되어 있어야 한다
- 상단에는 현재 스페이스 이름이 잘 보이게
- 하단 프리뷰 카드가 열려 있어도 좋지만, 첫 장은 지도 전체 인지가 우선

### 준비하면 좋은 예시 데이터
- 저장 장소 5~8개
- 방문 완료 2~3개
- 미방문 3~5개

---

## 페이지 2
### 역할
- "이 앱으로 무엇을 저장하는가"를 구체적으로 보여주는 장면

### 메인 카피
`가고 싶은 곳을 같이 모아두세요`

### 서브 카피
`장소 이름, 태그, 메모까지 남겨두면 데이트 버킷리스트가 완성돼요`

### 추천 화면
- `AddPlaceView`
- 또는 `PlaceDetailView`

### 화면 연출 가이드
- 태그와 메모가 잘 보이는 장소 상세 화면이 가장 적합
- 예시 메모는 감정이 느껴지는 문장으로 구성
  - 예: `야경 예쁠 때 같이 가기`
  - 예: `주말 브런치 후보`
- 장소 카테고리와 태그가 시각적으로 잘 드러나게

### 준비하면 좋은 예시 데이터
- 장소명: `성수 브런치 클럽`
- 태그: `#데이트`, `#브런치`, `#다음주`
- 메모: `테라스 자리 예쁘대. 날 좋을 때 같이 가자.`

---

## 페이지 3
### 역할
- 퐁당이 단순 저장 앱이 아니라 "추억 기록 앱"이라는 점 강조

### 메인 카피
`다녀온 순간은 기록으로 남겨요`

### 서브 카피
`방문 날짜, 별점, 한 줄 메모로 그날의 기억을 쌓아보세요`

### 추천 화면
- `VisitRecordFormView`
- 또는 `PlaceDetailView`의 방문 기록 목록

### 화면 연출 가이드
- 입력 화면을 보여주면 기능 전달이 직관적
- 이미 저장된 기록 리스트를 보여주면 결과물이 좋아 보임
- 가능하면 별점과 제목, 날짜가 동시에 보이게

### 준비하면 좋은 예시 데이터
- 제목: `비 오는 날이라 더 좋았던 카페`
- 별점: `5점`
- 본문: `창가 자리 분위기가 좋아서 오래 앉아 있었음`

---

## 페이지 4
### 역할
- 시간이 지나도 다시 돌아볼 수 있는 앱이라는 점 강조

### 메인 카피
`날짜별로 추억을 다시 꺼내보세요`

### 서브 카피
`캘린더에서 우리가 다녀온 날과 기록을 한눈에 볼 수 있어요`

### 추천 화면
- `HistoryCalendarView`

### 화면 연출 가이드
- 날짜 표시가 많은 달을 선택
- 특정 날짜를 눌렀을 때 기록 목록이 아래에 보이는 상태
- 월 이동 UI가 드러나면 더 좋음

### 준비하면 좋은 예시 데이터
- 한 달 안에 방문 기록 4~6개
- 같은 달 안에서 여러 날짜에 점 표시

---

## 페이지 5
### 역할
- "혼자 쓰는 앱이 아니다"를 가장 직접적으로 보여주는 장면

### 메인 카피
`스페이스로 함께 관리하세요`

### 서브 카피
`커플, 친구, 가족별로 공간을 나눠 장소와 추억을 같이 쌓을 수 있어요`

### 추천 화면
- `SpaceListView`
- `ManageSpaceView`
- `InviteView`

### 화면 연출 가이드
- 스페이스 목록이 2개 이상 보이면 다중 스페이스 지원 인지가 쉬움
- 또는 초대 코드/멤버 목록이 보이는 관리 화면도 좋음
- 이 장에서는 "함께 쓰는 구조"가 드러나는 것이 핵심

### 준비하면 좋은 예시 데이터
- 스페이스 예시:
  - `우리 둘`
  - `가족 나들이`
- 멤버 2~3명 표시

---

## 페이지 6
### 역할
- 실사용 편의성 강조

### 메인 카피
`길 찾기까지 자연스럽게 이어져요`

### 서브 카피
`저장한 장소를 카카오맵이나 네이버지도로 바로 열 수 있어요`

### 추천 화면
- `PlaceDetailView`
- 외부 지도 앱 열기 버튼이 보이는 상태
- 또는 `SettingsView`에서 기본 지도 앱 선택 화면

### 화면 연출 가이드
- 장소 상세 화면에서 외부 지도 버튼이 명확히 보이게
- 설정 화면을 함께 보여주고 싶다면 6장 대신 7장 구성으로 분리 추천

### 준비하면 좋은 예시 데이터
- 실제 주소가 있는 장소 1개
- 외부 지도 앱 버튼이 상단에서 잘 보이는 화면

---

## 선택 확장 페이지 7
### 역할
- 세부 설정과 생활 밀착감 보완

### 메인 카피
`내 취향에 맞게 설정해요`

### 서브 카피
`기본 지도 앱과 알림까지 필요한 방식으로 조정할 수 있어요`

### 추천 화면
- `SettingsView`

### 언제 넣으면 좋은가
- 기능형 앱 느낌을 조금 더 주고 싶을 때
- 스크린샷 6장만으로는 설정/완성도가 덜 보일 때

## 6장 구성 요약
| 페이지 | 핵심 메시지 | 추천 화면 |
|------|------|------|
| 1 | 함께 쓰는 장소 지도 | MapView |
| 2 | 가고 싶은 곳 저장 | AddPlaceView / PlaceDetailView |
| 3 | 방문 후 기록 남기기 | VisitRecordFormView / PlaceDetailView |
| 4 | 날짜별 추억 회고 | HistoryCalendarView |
| 5 | 스페이스로 함께 관리 | SpaceListView / ManageSpaceView / InviteView |
| 6 | 외부 지도 앱 연결 | PlaceDetailView / SettingsView |

## 미리보기 영상이 있다면
- 15~20초 내외 권장
- 흐름:
  1. 지도에서 장소 확인
  2. 장소 추가
  3. 메모 작성
  4. 방문 기록 작성
  5. 캘린더에서 다시 보기

## 실제 제작 시 주의사항
- 첫 장 카피가 가장 중요하다. 앱의 정체성을 바로 설명해야 한다.
- 기능 설명은 많아도 되지 않는다. 한 장마다 메시지는 1개만 남긴다.
- 예시 데이터 문구는 너무 기능 테스트처럼 쓰지 말고 실제 사용자가 남길 법한 말투로 구성한다.
- 지도 화면은 실제 생활권 느낌이 나도록 핀 배치를 자연스럽게 만든다.
- 스크린샷끼리 색감과 문장 톤을 맞춰야 앱 전체 완성도가 높아 보인다.
